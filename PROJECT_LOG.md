## 当前项目总览

- 项目类型：双碳能源管理与碳排放可视化平台。
- 当前形态：前端静态页面 + Java Spring Boot 后端 + MySQL 数据库。
- 当前定位：可运行的演示原型和后续正式系统的基础框架。
- 前端目录：`viewweb/`
- 后端目录：`src/main/java/`
- 数据库目录：`database/`
- 启动脚本：`run-java-backend.cmd`
- 检查脚本：`check-project.cmd`

## 当前完成度评估

- 按课程项目 / 演示原型看：约 65% - 70%。
- 按客户正式上线系统看：约 35% - 45%。

说明：现在已经能启动后端、连接 MySQL、管理员登录、维护区域和电耗、展示部分数据库图表；但正式交付还需要补权限、安全、批量导入、测试、部署和更多真实业务模块。

## 当前已完成能力

- Java Spring Boot 后端已经搭建完成，Node.js 原型后端已经清理。
- MySQL 数据库结构已经建立，包含组织、区域、能源类型、表计、能源读数、预算、用户、角色、审计日志、预警规则等表。
- `.env` 已用于保存数据库连接配置，数据库密码不写入代码。
- 项目已内置便携版 JDK 17 和 Maven，可以不依赖系统 PATH 直接构建运行。
- 首页已经接入 MySQL 汇总数据。
- 管理员登录入口已放到首页右上角。
- 管理员后台已支持登录校验、新增区域、编辑区域、停用区域、录入电耗、修改管理员密码。
- 后端已提供审计日志查询接口，碳核查支撑页可优先读取真实 `audit_logs` 留痕。
- 默认管理员账号为 `admin / 123456`。
- 区域查看页可以查看区域和用电汇总数据。
- `能源查询`、`预算管理`、`碳排放管理` 三个模块已经优先读取 MySQL 汇总数据。
- 管理员录入电耗后，会同步写入 `electricity_readings` 和通用能源表 `energy_readings`。
- `seed.sql` 已补充 2025 年 12 个月演示电耗数据，方便真实数据未到前演示趋势图。
- `README.md` 已更新当前启动、数据库、接口和状态说明。

## 当前主要不足

- 还有部分模块仍然使用前端模拟数据，例如供应链、核查、碳资产、能流、能效分析等。
- 管理员权限是轻量实现，token 存在后端内存中，后端重启后登录状态会失效。
- 管理员后台功能还不完整，暂时没有用户管理、角色管理、表计管理、排放因子维护、预算维护、审计日志页面。
- 还没有 Excel / CSV 批量导入功能，后续真实大量数据接入会不方便。
- 数据库升级方式仍然依赖手动执行 `schema.sql` 和 `seed.sql`，正式项目建议使用 Flyway 或 Liquibase。
- 前端仍是原生 HTML/CSS/JS，模块继续变多后维护成本会升高。
- 后端还没有自动化测试。
- 当前部署方式偏开发环境，正式交付需要 Windows 服务、Docker 或一键安装启动方案。
- 默认密码 `123456` 和 `CORS_ORIGIN=*` 只适合本地开发或演示，不适合正式环境。
